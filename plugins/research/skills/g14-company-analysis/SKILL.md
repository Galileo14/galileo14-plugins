# g14-company-analysis

Eleven platform-specific researcher agents fan out in parallel, one synthesizer fills a branded HTML template, two fresh graders verify quality.

## Architecture

This skill is a thin orchestrator. **All knowledge lives in plugin-level agents.** The skill captures input, manages folders, fires agents in parallel, and assembles the final deliverable.

| Component | Owner | Role |
|---|---|---|
| Main agent (you) | this skill | Capture input, create slug + folders, orchestrate fan-outs, apply verdicts, present result |
| 11 platform researchers | `${CLAUDE_PLUGIN_ROOT}/agents/*-researcher.md` (and `website-crawler.md`) | One per platform — each writes 1 findings file |
| `research-synthesizer` | plugin agent | Fills the HTML template with the 11 findings |
| `hallucination-grader` + `source-validator` | plugin agents | Fresh-context quality gates, run in parallel |

All agents use Haiku (set in their frontmatter). The skill just fires them.

## Flow (6 phases)

### Phase 1 — Capture the company

- If the user hasn't provided the name, ask in a single line. If they haven't provided the web domain, try to infer it from the first sentence and confirm in one line (without researching anything yet).
- Generate a `slug` in kebab-case from the trade name. No accents, no punctuation, max 5 words. E.g.: "Grupo Billingham" → `grupo-billingham`.
- Detect the user's language; the final report will be in that language. Default to **English** if unclear.
- Create in the CWD:

```
research/{slug}/
  findings/             # 11 files generated by platform researchers
  {slug}.html           # final report (written by research-synthesizer)
  _grader-reports/      # JSON from hallucination-grader and source-validator
```

If the folder already exists (resume after interruption), see "Resume" at the end.

### Phase 2 — Launch the 11 platform researchers IN PARALLEL (single message)

**Critical:** the 11 `Task` tool calls go in the **same assistant turn**, in a single block. If you split them across separate messages, they run serially and the skill loses its purpose.

Map of dimension number → agent name → output filename:

| # | Agent | Output file |
|---|---|---|
| 01 | `website-crawler` | `findings/01-website.md` |
| 02 | `google-researcher` | `findings/02-google.md` |
| 03 | `linkedin-researcher` | `findings/03-linkedin.md` |
| 04 | `reddit-researcher` | `findings/04-reddit.md` |
| 05 | `trustpilot-researcher` | `findings/05-trustpilot.md` |
| 06 | `google-maps-researcher` | `findings/06-google-maps.md` |
| 07 | `glassdoor-researcher` | `findings/07-glassdoor.md` |
| 08 | `indeed-researcher` | `findings/08-indeed.md` |
| 09 | `youtube-researcher` | `findings/09-youtube.md` |
| 10 | `news-researcher` | `findings/10-news.md` |
| 11 | `crunchbase-researcher` | `findings/11-crunchbase.md` |

For each, invoke `Task` with `subagent_type: <agent-name>` and a prompt that supplies:
- `target_name: {company_name}`
- `target_domain: {company_domain}`
- `output_path: {absolute path from the table above}`
- `language: {detected language}`

The agents own the methodology (queries, URLs to visit, signals to extract, anti-hallucination rules, findings file structure). You don't repeat any of that here.

When all 11 finish, verify the 11 files exist in `research/{slug}/findings/`. If any is missing, **relaunch only that agent** with the same prompt.

### Phase 3 — Synthesis into HTML template (1 call)

Invoke `research-synthesizer` (in a single message) with:
- `format: html-template`
- `template_path: ${CLAUDE_PLUGIN_ROOT}/skills/g14-company-analysis/assets/report-template.html`
- `findings_paths: [absolute paths to all 11 findings files]`
- `output_path: research/{slug}/{slug}.html`
- `topic: {company_name}`
- `language: {detected language}`
- `slug: {slug}`
- `extra_context: company_name={name}, company_domain={domain}`

The agent fills every `{{SLOT}}` from the findings, handles the linking rules (URL extraction + the deterministic Google Maps address transformation), and leaves the yellow banner out (the orchestrator decides that in Phase 5).

### Phase 4 — Quality gate IN PARALLEL (single message)

**Critical:** the 2 grader `Task` calls go in the same turn, both fresh contexts (not the synthesizer).

In one message, fire both:

**Hallucination grader** — `Task` with `subagent_type: hallucination-grader` and prompt:
- `output_path: research/{slug}/{slug}.html`
- `findings_dir: research/{slug}/findings/`
- `verdict_path: research/{slug}/_grader-reports/hallucination.json`

**Source validator** — `Task` with `subagent_type: source-validator` and prompt:
- `output_path: research/{slug}/{slug}.html`
- `verdict_path: research/{slug}/_grader-reports/reachability.json`

Both agents return JSON verdicts and also write them to disk. Read both JSONs from disk to apply the verdict in Phase 5.

### Phase 5 — Apply verdict and annotate the HTML

With both JSON files in hand, decide:

**Case A — Clean pass:**
- `hallucination`: `pass >= 13/15`
- `reachability`: `(ok + ok_gated) / urls_evaluated >= 90%`

→ Deliver as-is. No banner. No marks.

**Case B — Pass with warnings:**
- `hallucination`: `pass between 10-12/15`
- Or `reachability`: between 70-89%

→ Edit the HTML with the `Edit` tool:
1. **Uncomment the banner** the synthesizer left wrapped: remove the two marker lines `<!-- BANNER-OFF` and `BANNER-OFF -->` so the banner block becomes live HTML. Then replace `{{BANNER_MESSAGE}}` inside it with text like: *"{N} of 15 reviewed claims could not be traced to their source; {M} cited URLs are unreachable. Review before acting on the marked sections."*
2. For each claim with verdict `FAIL_*` or `UNVERIFIED` from `hallucination-grader`: wrap with `<mark data-unverified="true">{text}</mark>` wherever it appears in the HTML.
3. For each URL with verdict `DEAD`, `REDIRECTED`, `UNREACHABLE`, or `DOMAIN_MISMATCH` from `source-validator`: add `class="source-warning"` to the corresponding `<a>`.

**Case C — Insufficient quality:**
- `hallucination`: `pass < 10/15`
- Or `reachability`: < 70%

→ **Do not deliver.** Report to the user:

```
⚠ Insufficient report quality — not delivered.

- Traceability: {X}/15 claims confirmed.
- Source reachability: {Y}% of URLs valid.

Worst dimensions: {list of findings with most failures}.

Options:
1. Relaunch synthesis (may help if it was a synthesizer issue).
2. Relaunch problematic platform researchers: {list}.
3. Abort.

Which?
```

### Phase 6 — Present to the user

```
✅ Company analysis complete: **{COMPANY_NAME}**

📄 Report: `research/{slug}/{slug}.html`
🔍 11 dimensions researched in parallel · {N total} sources consulted
✔ Verification: {pass}/15 claims traced · {pct}% URLs reachable

**Executive summary:**
{first paragraph of the executive summary from the HTML}

**Most relevant signals:**
- {1 cross-signals bullet}
- {1 cross-signals bullet}
- {1 cross-signals bullet}

Would you like me to go deeper on any dimension, export to PDF, or research a competitor?
```

## Critical rules

- **Parallelism in a single message.** The 11 platform-researcher `Task` calls in Phase 2 go together. The 2 graders in Phase 4 also together. That's the difference between 30 seconds and 5 minutes.
- **Agents own knowledge.** Do not repeat platform-specific queries, signals, or rubrics inside this skill. If a researcher needs improving, edit the agent.
- **The HTML template is authoritative.** The synthesizer doesn't invent structure — only fills slots. Do not modify the template ad-hoc.
- **Graders are fresh.** Never reuse the synthesizer's context for grading — the plugin's agent definitions ensure this when invoked separately.
- **Do not read the findings in the main agent.** The synthesizer and graders do that. You only verify that files exist and orchestrate.
- **Report language** = user's language. Default to English if unclear.

## Quick troubleshooting

- **A platform researcher didn't write its findings:** relaunch only that agent with the same prompt.
- **Multiple researchers fail at once:** probable rate limit. Wait 1-2 min and relaunch only the failed ones.
- **Synthesizer produces truncated HTML or unfilled slots:** retry emphasizing "fill EVERY slot, leave no `{{...}}`". If it keeps failing, escalate that single call to `sonnet` (override the agent's model just for the retry).
- **Hallucination grader reports many FAIL_NO_TRACE:** the synthesizer is hallucinating — relaunch Phase 3 (consider sonnet retry).
- **Source validator reports many DEAD URLs:** platform researchers are citing unverified URLs — relaunch the dimensions with the worst ratio.
- **Very small / niche company:** many platforms will return "no presence". Legitimate. The final report will be shorter but equally valid.

## Resume after interruption

1. List `research/{slug}/findings/`. If fewer than 11 files, identify missing ones by absent numeric prefix (`01-`, `02-`, …, `11-`).
2. Relaunch only the platform researchers for the missing dimensions.
3. If all 11 findings exist but `{slug}.html` does not, launch only the synthesizer (Phase 3).
4. If the HTML exists but the grader JSONs in `_grader-reports/` do not, launch only Phase 4.
